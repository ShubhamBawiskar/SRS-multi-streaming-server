Automated RTMP Streaming Server

A professional guide to using SRS and FFmpeg for dynamic video stream management.

Project Goal

  • Accept multiple input sources (camera RTSP/RTMP, local videos, etc.).
  • Restream them through SRS as dynamic RTMP endpoints (e.g., rtmp://your-ip/live/cam1).
  • Automate the entire setup to avoid manual configuration for each new stream.

Technology Stack

  • SRS (Simple Realtime Server): The core RTMP server.
  • FFmpeg: For ingesting, re-encoding, or relaying inputs to RTMP.
  • Python Script: To automate adding and managing new sources.
  • Docker: For easy and clean SRS deployment.

System Architecture

This setup creates a flexible pipeline. The Python script acts as a manager, using FFmpeg to pull video from various sources and push it to the SRS media server. SRS then makes these streams available to viewers on demand.

Implementation Guide

Step 1: Install Prerequisites

Ensure Docker and FFmpeg are installed on your system. Docker will run the SRS server, and FFmpeg is required by the Python script.

  • Install Docker
  • Install FFmpeg

Step 2: Launch the SRS Media Server

Use the recommended Docker command to start an instance of SRS. This command maps the necessary ports for RTMP, the API, and the web console.

docker run -it -d --name srs \
  -p 1935:1935 -p 1985:1985 -p 8080:8080 \
  ossrs/srs:latest

Step 3: Configure and Run the Python Script

Save the Python script below as a file (e.g., `start_streams.py`). Modify the `streams` dictionary to include your actual video sources and desired stream keys.

import subprocess
import time

# Define your input sources and desired output stream keys
streams = {
    "cam1": "rtsp://192.168.1.10/stream1",
    "cam2": "rtsp://192.168.1.11/stream2",
    "video1": "/path/to/your/local/video1.mp4"
}

def start_stream(name, source):
    output_url = f"rtmp://localhost:1935/live/{name}"
    command = [
        "ffmpeg", "-re", "-i", source,
        "-c:v", "copy", "-c:a", "aac", "-f", "flv",
        output_url
    ]
    print(f"ðŸš€ Starting stream '{name}'...")
    return subprocess.Popen(command)

processes = []
for name, src in streams.items():
    processes.append(start_stream(name, src))
    time.sleep(2)

print("\nâœ… All stream processes initiated.")
try:
    for p in processes: p.wait()
except KeyboardInterrupt:

    for p in processes: p.terminate()

Run the script from your terminal:

python start_streams.py

Step 4: Access and View the Streams

Your streams are now live. Use an RTMP-compatible player to view them. Replace `` with the IP address of the machine running Docker.

rtmp://<your-server-ip>/live/cam1

rtmp://<your-server-ip>/live/cam2

You can preview them in:

  • VLC Media Player: Go to Media > Open Network Stream...
  • Web Player: Use a player like video.js or Clappr to embed the stream.
